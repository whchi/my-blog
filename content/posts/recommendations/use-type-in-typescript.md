# Summary
影片中的案例是大量亂用 interface 定義型別，正常情況下還是以語意為基礎：**interface 用於定義物件的結構，type 用於定義其他型別。**

1. type 比起 interface 更為簡潔
2. 定義複雜型別時，type 比起 interface 更為易懂
3. 同名的 interface IDE 不會提示

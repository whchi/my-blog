# Summary
很多很理想的金句，可以當雞湯喝一喝

1. 傳統開發流程的假設已失效

寫程式之前最難因此需要大量 ticket 拆 sprint / 估點數，目的在彌補 intent 與 implementation 之間的 gap，AI 直接 intent -> implement -> review -> ship

2. 瓶頸翻轉

從「規劃與協調」轉移到「審查與評估」，重點不在是 what / who to do，而是評估 AI 產出是否正確、符合架構、少 bug、易於維護

4. 資深工程師的價值從「事前」轉移到「事後」

以前是寫文件拆任務，現在是判斷品味

5. 哪些會死亡/減少
   1. story points
   2. sprint planning
   3. blacklog grooming
   4. project coordination layer
5. 哪些會成長/轉型
   1. 架構判斷力
   2. code review 品質評估
   3. intent 表達能力
   4. 系統思考與整合力

6. Artchitectural Decision Records(ADRs) 是核心

「為什麼要這樣設計」是有意義的問題，對 agent 而言是約束，對 reviewer 而言是判斷的框架
